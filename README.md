OMNIA アナリティクス・データ解析プロジェクト集
本プロジェクトは、Google Colabおよびローカル環境において、デリバリーサービスの注文データや配送ログを解析し、マーケティング施策の評価やオペレーション上のボトルネックを特定するためのスクリプト群です。

📁 フォルダ構成（DataStorage）
各スクリプトは、以下のデータ構造を前提にパスを自動解決します。

Plaintext
データ集計・DataStorage/
├── join_orders/                       # 注文結合データ (TSV)
├── order_data/                        # 注文マスタ (CSV)
├── delivery_status_history_data/      # 配達ステータス履歴 (CSV)
└── driver_shop_arrival_history_data/  # ドライバー店舗到着時刻 (CSV)
🛠 1. 注文データ分析 & 還元祭クーポン算出
join_orders データを活用し、ユーザー抽出、購入率算出、およびキャンペーンリスト作成を行います。

主な解析内容
施策ユーザー抽出: 施策A・Bの重複・単発購入者を集合演算で分類し、コストを試算。

既存購入率・定価購入率: 既存ユーザーの定着度および、クーポンに頼らない「定価購入」の割合を算出。

還元祭クーポン算出: 特定期間のデリバリー注文を抽出し、実支払額の30%還元ロジックに基づいて付与リストを作成。

エリア別集計: ユーザー位置（ユーザーチーム）と店舗位置（クルーチーム）の2軸で、デイリー平均注文数を集計。

🚚 2. 配送遅延要因分析 (Case Study 2)
3つのデータを統合し、承認から完了までの工程を分解することで**「遅延の真因」**を特定します。

工程定義（一本道フロー）
遅延分析では、工程を重複のない4フェーズに分解して評価します。

受諾待ち: 承認 ～ クルー受諾（マッチング時間）

クルー移動: 受諾 ～ 店舗到着（クルーの移動時間）

店舗待機: 到着 ～ ピックアップ（調理・受け渡し時間）

最終配送: ピックアップ ～ 完了（ユーザーへの配送時間）

主な機能
ボトルネック判定: 遅延注文ごとに「4工程のうち最も時間を要した工程」を自動特定。

データクレンジング: 時刻逆転（負の値）の自動除外、UNIX秒の変換、重複排除を実装。

多角的な統計指標: 平均・中央値に加え、ワーストケース把握のための p90（上位90%タイル） を算出。

マルチ出力: 解析結果を Excel（.xlsx）形式で保存し、Google スプレッドシートへ自動書き込み。

🚀 実行環境と準備
必須ライブラリ
Bash
pip install pandas openpyxl gspread google-auth pathlib
パス解決ロジック
スクリプト内には、Google Drive（Colab）とローカルのWindowsパス（Gドライブ等）を自動で切り替える機能が備わっています。実行環境に合わせて以下のパスが優先的に探索されます。

/content/drive/MyDrive/... (Colab)

G:\マイドライブ\... (Local Windows)

💡 注意事項
日付フィルタ: キャンペーン初日の「11時開始」など、時刻指定を含むフィルタリングを行っているため、期間変更時はコード内の order_dt 指定箇所を確認してください。

予約注文: time_type=2（予約）は、即時注文とリードタイムの性質が異なるため、分析目的に応じて除外またはフラグ分けを行っています。
