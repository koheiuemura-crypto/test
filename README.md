🛠 1. 注文データ分析 & 還元祭クーポン算出
join_orders データを主軸に、ユーザー行動の可視化と施策用のインセンティブ算出を行います。

🔹 実施内容
施策別ユーザー抽出とコスト試算:

施策A・Bの重複購入者、単発購入者を集合演算で分類。

金額条件（300円/500円/750円）に基づいた総コストの自動計算。

既存ユーザー分析:

既存購入率：既存母数に対する当日ユニーク購入者数の割合。

定価購入率：デリバリー注文かつ既存ユーザーにおける「クーポン不使用」注文の割合。

還元祭クーポン (Repay Festival) ロジック:

条件：1,500円以上、デリバリー、完了済み、即時注文。

特徴：「配達料無料クーポン」併用時は商品価格ベース、非併用時は実支払額ベースで30%還元額を算出。

エリア別集計:

ユーザー位置（User Team）と店舗位置（Crew Team）の2軸でデイリー平均注文数を算出。

🚚 2. 配送遅延要因分析 (Case Study 2)
「承認から完了まで」の全工程を分解し、ボトルネックを特定します。

🔹 配送工程の定義（一本道モデル）
受諾待ち: 店舗承認 ～ クルー受諾（マッチング効率の指標）

クルー移動: 受諾 ～ 店舗到着（クルーの稼働効率）

店舗待機: 到着 ～ 商品受取（調理時間・受け渡し効率）

最終配送: 商品受取 ～ 配達完了（ラストワンマイルの配送効率）

🔹 分析機能
ボトルネック自動判定: 遅延が発生した注文に対し、4工程のうち最も時間を要した区間を特定。

統計指標 (P90指標): 平均値だけでなく、ワースト10%の状態を示す「p90」を算出し、サービス品質の底上げに活用。

異常値検知: 時刻の逆転（負の値）や極端な長時間滞留（AP149X等）をクレンジング。

🚀 実行環境
言語: Python 3.x

主要ライブラリ: pandas, pathlib, openpyxl, gspread

動作確認: Google Colab / Local Windows (Cursor / VS Code)

実行のポイント
パス解決: スクリプトは環境を自動判別し、ColabのドライブパスまたはローカルのGドライブパスを優先的に読み込みます。

ファイル出力: output/ フォルダに repay_festival_coupons.csv や遅延分析結果（Excel）が自動生成されます。
