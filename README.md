OMNIA アナリティクス・データ解析プロジェクト集
本プロジェクトは、Google Colabおよびローカル環境（Windows等）の両方で動作し、デリバリーサービスの各種データを統合解析するためのスクリプト群です。

📁 ディレクトリ構造 (DataStorage)
スクリプトは以下のパス構造を前提に設計されています。環境（Colab/Local）に応じたパスの自動解決機能を備えています。

🛠 1. 注文データ分析 & 還元祭クーポン算出
join_orders を主軸とし、マーケティング施策の評価とキャンペーン管理を行います。

🔹 主な解析内容
施策ユーザー抽出: 集合演算（Set）を用いて、施策A・Bの重複購入者や単発購入者を分類し、配布コストを試算。

既存購入率・定価購入率: 既存ユーザーの定着度、およびクーポン不使用の「定価購入」の割合を特定。

還元祭クーポン算出:

条件（1,500円以上、デリバリー限定、完了済、即時注文等）に合致する注文を抽出。

「配達料無料クーポン」の有無に応じた還元額計算（実支払額の30%）。

CSV出力（user_id, discount_price, discount_code）。

エリア別集計:

ユーザー視点: user_delivery_area_id 軸。

クルー視点: shop_delivery_area_id 軸。

各エリアのデイリー平均注文数を算出。

🚚 2. 配送遅延要因分析 (Case Study 2)
「どこで、なぜ配送が遅延しているのか」を特定するため、3つのマスタデータを統合し、配送工程を分解・可視化します。

🔹 工程フローの定義（一本道直列モデル）
遅延のボトルネックを特定するため、重複のない4つのフェーズを定義しています。


工程番号,フェーズ名,区間定義,分析の狙い
1,受諾待ち,承認 ～ クルー受諾,マッチング効率・クルー不足の可視化
2,クルー移動,受諾 ～ 店舗到着,クルーの走行距離・移動効率の評価
3,店舗待機,到着 ～ ピックアップ,店舗の調理遅延・受け渡し効率の評価
4,最終配送,ピックアップ ～ 完了,配送ルート・ラストワンマイルの評価
