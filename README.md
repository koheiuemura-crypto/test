OMNIA アナリティクス・データ解析プロジェクト本リポジトリは、デリバリーサービス「menu」のデータを活用した、マーケティング施策評価および配送オペレーション分析のためのスクリプト群を格納しています。📊 プロジェクト概要データドリブンな意思決定を支援するため、以下の2つの主要な分析機能を備えています。注文・施策効果分析: 施策コストの試算、既存顧客の購入率推移、キャンペーン対象者の自動抽出。配送遅延要因分析: 配送プロセスのボトルネック特定とサービス品質（SLA）の可視化。📁 フォルダ構成 (DataStorage)スクリプトは以下のディレクトリ構造を前提として動作します。Google Drive（Colab）およびローカル環境の両方でパスを自動解決します。Plaintextデータ集計・DataStorage/
├── join_orders/                       # 注文結合データ (TSV)

└── output/                            # 解析結果・CSV出力先
🛠 各スクリプトの機能1. 注文データ分析 & 還元祭クーポン算出join_orders をベースに、ビジネスKPIの算出とキャンペーン運用を自動化します。施策ユーザー抽出: 集合演算を用い、施策A・Bの重複購入者や単発購入者を瞬時に分類し、期待コストを試算。既存購入率分析: 既存ユーザー母数に対する当日アクティブ率や、クーポンに依存しない「定価購入率」を算出。還元祭ロジック: 特定期間（2021/9/25-28）のデリバリー注文に対し、決済種別に応じた30%還元額を算出。エリア別デイリー集計: ユーザーチーム（配送先）とクルーチーム（店舗所在地）それぞれの視点で、エリア別の1日平均注文数を集計。2. 配送遅延要因分析 (Case Study 2)複数の履歴データを統合し、配送完了までの工程を「直列4フェーズ」に分解してボトルネックを可視化します。工程名定義分析のポイント受諾待ち承認 ～ クルー受諾マッチング効率・車両不足の判定クルー移動受諾 ～ 店舗到着クルーの走行距離・配置の妥当性店舗待機到着 ～ ピックアップ調理遅延・店舗での受け渡し効率最終配送ピックアップ ～ 完了配送ルートの最適化・サービス品質ボトルネック判定: 遅延が発生した全注文に対し、上記4工程のうち「最も時間を要した工程」を特定。異常値検知: ログの逆転（負の値）や、極端な異常値（AP149X等）を自動検出し、統計から除外。多角的指標: 平均値だけでなく、ワーストケース（p90）を算出することで、サービス品質の底上げを支援。🚀 実行環境動作確認済み環境Google Colab: Google Driveを /content/drive にマウントして実行。Local (Windows/macOS): pathlib により、共有ドライブ等のパスを自動認識。必要ライブラリBashpip install pandas openpyxl gspread google-auth
💡 技術的な実装のポイントパスの動的解決: Path(__file__) や exists() を組み合わせ、環境を選ばずに動作する設計。データ整合性チェック: to_numeric による型変換と、欠損値（NaN）の適切な処理。マルチフォーマット出力: 解析結果は、分析用の CSV/Excel だけでなく、レポート共有用の Google スプレッドシート への自動書き込みにも対応。📝 修正履歴・特記事項工程定義の最適化: 以前の重複工程を廃止し、定義書に基づいた「一本道フロー」へ修正。予約注文の考慮: time_type を判別し、予約注文による待機時間を配送遅延と誤検知しないようロジックを強化。
